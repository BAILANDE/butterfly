在提问之前

在我之前的文章《全球AI狂潮，普通人何去何从》中，我曾提到过“初具人形、毫无学习能力且没有请教态度、只会伸手抓香蕉的猴子”。这次，我想仔细谈谈我认为提问前必须注意的事项。

在提问前，请先确保你已经尝试过以下操作：

  1. 阅读相关文档或官方手册。
  2. 使用搜索引擎查询类似问题。
  3. 利用 AI 工具（如 ChatGPT）辅助查找答案。
  4. 自己动手尝试解决问题，并记录尝试过程和结果。

在以上方法均无法解决问题时，再进行提问，并尽量提供完整信息。

编辑问题

保持问题完整

一个高质量的问题至少应包含 “背景 + 目标 + 现状 + 尝试 + 初步判断 + 补充信息 + 态度”。

比如，以下是不完整的问题示例：

hexo博客部署失败咋整？

而完整且清晰的提问可以这样写：

我在用 Hexo 搭建博客**（背景），
想部署到 GitHub Pages 上（目标），
但执行 hexo deploy 时失败（现状）**，

已经配置了 _config.yml 并设置了仓库地址**（尝试），
我怀疑是网络问题（初步判断），
报错信息 xxxx，Windows 版本 xxxx，Hexo 版本 xxxx（补充信息），
请问大家可能是什么问题？（态度）**

如果能附上你自己的判断会更好，但即便没有，也没关系——毕竟你是在请教问题，没有人会嘲笑你没有自己的判断。

不要发语音

提问尽量用文字，而不是语音。原因如下：

  1. 便于快速理解
    文字信息可以被快速浏览、定位重点，而语音需要完整听完才能抓住问题核心。
  2. 方便记录和检索
    文字可以复制、保存或搜索，便于复查和参考；语音无法直接检索。
  3. 利于多人协作
    在社区或团队中，文字提问让多人同时理解问题，方便讨论；语音通常限制于一对一交流。
  4. 提升提问质量
    写文字时，你会整理思路、列出背景和尝试，这本身就是对问题的梳理；语音容易跳跃、逻辑混乱，别人难以准确回答。
  5. 尊重他人时间
    回答问题大多出于善意，文字让别人快速判断是否能回答，而语音增加理解成本，可能降低回应意愿。

提问之后

  1. 记录关键步骤
    将有效方法和操作步骤整理成笔记，以便后续参考。
  2. 反馈结果
    无论问题是否解决，都要回复回答者结果，并表达感谢。

我只是在这里浅显的提到提问的方式，我更建议读一读这篇文章：How-To-Ask-Questions-The-Smart-Way会非常有帮助。
